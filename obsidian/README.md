# Jarvis-Pro Obsidian Vault Structure

## PARA Architecture

This vault follows the PARA method adapted for an SEO agency:

```
0-Inbox/                    ← Capture zone; process daily
1-Projects/                 ← Active, time-bounded work
│   └── {client-id}/        ← One folder per active project
2-Areas/                    ← Ongoing responsibilities
│   ├── Clients/            ← Client profile notes (auto-synced)
│   └── SEO-Research/       ← Ongoing research areas
3-Resources/                ← Reference material
│   ├── Research/           ← Auto-saved research harness output
│   └── Tool-Docs/          ← Integration guides, API docs
4-Archive/                  ← Completed work
Maps/                       ← Maps of Content + dashboards
_Templates/                 ← Note templates (do not edit directly)
```

## Key Notes

- **Client profiles** (`2-Areas/Clients/{id}.md`) are auto-generated by `jarvis obsidian sync-clients`
  and contain a Dataview query to surface all related notes.
- **Research notes** (`3-Resources/Research/`) are auto-written by the research harness.
  Each note has frontmatter linking it to a client and includes source citations.
- **Skill output** (`1-Projects/{client-id}/`) is written when you run a skill with `--save-vault`.
- **Weekly dashboard** (`Maps/Weekly-Dashboard-{year}-W{week}.md`) is rebuilt by `jarvis obsidian dashboard`.

## Recommended Obsidian Plugins

| Plugin | Purpose |
|--------|---------|
| Dataview | Dynamic queries in client profiles and dashboards |
| Templater | Advanced template processing |
| Calendar | Navigate daily/weekly logs |
| Tag Wrangler | Manage the tag taxonomy |
| Obsidian Git | Auto-commit vault to git |

## Tag Taxonomy

```
#client/{id}        - Notes about a specific client
#research           - Research harness output
#action-needed      - Needs follow-up
#weekly-report      - Weekly report notes
#citation-audit     - Citation audit outputs
#falcon-report      - Local Falcon data
#gbp                - Google Business Profile
```

## Dataview Snippets

### All active clients with last-updated date
```dataview
TABLE primary_keyword AS "Primary KW", city, active
FROM "2-Areas/Clients"
WHERE active = true
SORT file.name ASC
```

### Notes needing action
```dataview
LIST
FROM ""
WHERE contains(tags, "action-needed")
SORT file.mtime DESC
```

### Recent research
```dataview
TABLE client, date
FROM "3-Resources/Research"
SORT date DESC
LIMIT 20
```
